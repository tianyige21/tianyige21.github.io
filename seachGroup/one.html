# 一. vue3 和 vue2 的区别？

  1. 性能优化：Vue 3 在内部进行了大量的性能优化，包括更快的渲染速度和更小的包大小。这主要是通过使用 Proxy 代替 Object.defineProperty 实现的。
  2. Composition API：Vue 3 引入了 Composition API，使开发者可以更方便地组织和重用代码。Composition API 通过提供一组逻辑相关的函数，允许将逻辑分组成功能更强大的逻辑单元。
  3. Teleport：Vue 3 引入了 Teleport 组件，它允许您将内容渲染到 DOM 中的任何位置，这对于处理模态对话框、下拉菜单等非常有用。
  4. 新的响应式系统：Vue 3 的响应式系统与 Vue 2 相比有很大的变化。它使用 Proxy 代理实现，这使得它比 Vue 2 的响应式系统更快、更强大。
  5. 改进的 TypeScript 支持：Vue 3 具有更好的 TypeScript 支持，包括更好的类型推断和更好的类型声明。

总的来说，Vue 3 与 Vue 2 相比有许多改进和新功能，这使得它更加强大、更加灵活，同时也更容易开发和维护。

# 二. webpack 使用什么插件使得 css 和 js 分离？

在 Webpack 中，可以使用 mini-css-extract-plugin 插件将 CSS 从 JavaScript 中分离出来。这个插件可以将每个 CSS 文件都提取为一个单独的文件，并使用 标签将其插入到 HTML 中，从而避免了在页面加载时将所有样式打包到 JavaScript 中。

  • # mini-css-extract-plugin

const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist',
  },
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [MiniCssExtractPlugin.loader, 'css-loader'],
      },
    ],
  },
  plugins: [
    new MiniCssExtractPlugin({
      filename: 'style.css',
    }),
  ],
};

上面的配置将 MiniCssExtractPlugin 插件用于提取 CSS 文件，并将所有 CSS 文件打包成一个单独的 style.css 文件。在 HTML 文件中使用 标签引入此文件即可。

需要注意的是，这个插件只能将 CSS 文件从 JavaScript 中分离出来，如果需要将 JavaScript 文件从 HTML 中分离出来，可以使用 html-webpack-plugin 插件。

  • # html-webpack-plugin 插件。

html-webpack-plugin 是一个 Webpack 插件，用于将生成的 JavaScript 文件自动注入到 HTML 文件中，并将该文件生成到指定的目录中。

在 Webpack 配置中使用 html-webpack-plugin 插件时，可以通过指定模板文件和输出文件名来自动生成 HTML 文件，并将生成的 JavaScript 文件注入到该文件中。以下是一个示例配置：

const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist',
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
      filename: 'index.html',
    }),
  ],
};

上面的配置中，指定了 template 选项为 ./src/index.html，表示使用该文件作为 HTML 文件的模板。指定 filename 选项为 index.html，表示生成的 HTML 文件将保存在 dist 目录下，并命名为 index.html。

当 Webpack 运行时，html-webpack-plugin 插件会自动读取指定的模板文件，并将生成的 JavaScript 文件注入到该文件中，最终生成一个包含注入的 JavaScript 文件的 HTML 文件。

# 三. 请解释一下 node 的事件流

Node.js 中的事件流（Event Loop）是一种异步 I/O 模型，用于处理事件和回调函数。事件流是一个循环，不断地从事件队列中获取事件并执行相应的回调函数。以下是事件流的工作原理：

  1. 事件循环：事件循环是 Node.js 中的核心机制，用于管理事件的生命周期。它会不断地检查事件队列中是否有新的事件需要处理。

  2. 触发事件：当一个异步操作完成时，Node.js 会将事件放入事件队列中。事件队列是一个先进先出的队列，保存了所有需要处理的事件。

  3. 选择回调函数：事件循环会从事件队列中取出一个事件，并查找与该事件关联的回调函数。每个事件都有一个或多个回调函数，这些函数会在事件被处理时被调用。

  4. 执行回调函数：事件循环会执行选定的回调函数，并传递相应的参数。这些回调函数通常是异步的，因此它们不会阻塞事件循环。在回调函数执行期间，事件循环会暂停，等待回调函数执行完毕。

  5. 重复循环：当回调函数执行完毕后，事件循环会重新开始循环，检查是否有新的事件需要处理。这个过程会一直重复，直到事件队列中没有事件为止。

总之，事件流允许 Node.js 处理大量的并发请求，并保证在所有的回调函数被调用之前，事件队列中的所有事件都能得到处理。这种模型的优点是它能够有效地利用计算资源，并避免阻塞程序的运行。

# 四. module.export 和 exports 有什么区别？

当在 Node.js 中编写模块时，经常会看到两个对象：module.exports 和 exports。这两个对象的作用相同，但是在使用时需要注意它们之间的区别。以下是它们的区别：

  • module.exports 是模块的默认输出，如果一个模块没有指定 exports 属性，则默认输出为 module.exports 对象。
  • exports 是 module.exports 的一个辅助对象，它可以向外部暴露模块中的函数或变量。可以将需要导出的函数或变量添加到 exports 对象中，使其对外可见。

下面是一个简单的示例，展示了如何使用 module.exports 和 exports：

// greet.js
module.exports = function (name) {
  console.log(`Hello, ${name}!`);
};

exports.goodbye = function (name) {
  console.log(`Goodbye, ${name}!`);
};

在上面的示例中，module.exports 导出了一个函数，而 exports 导出了另一个函数。当需要将模块作为整体导出时，应该使用 module.exports。如果需要向外部暴露多个函数或变量，则可以使用 exports。

需要注意的是，不能直接将 exports 赋值为一个新对象，因为这会破坏 exports 和 module.exports 之间的联系。如果需要将 exports 赋值为一个新对象，则应该使用 module.exports 来导出整个模块。以下是一个示例：

// error.js
exports = {
  notFound: function (msg) {
    return new Error(`Not found: ${msg}`);
  },
  invalidArg: function (msg) {
    return new Error(`Invalid argument: ${msg}`);
  },
};

// 这个模块将无法正常工作，因为 exports 和 module.exports 不再指向同一个对象

正确的写法应该是：

// error.js
module.exports = {
  notFound: function (msg) {
    return new Error(`Not found: ${msg}`);
  },
  invalidArg: function (msg) {
    return new Error(`Invalid argument: ${msg}`);
  },
};

// 这个模块可以正常工作

总之，module.exports 和 exports 都可以用于向外部暴露模块中的函数或变量，但是它们之间有一些细微的差别，需要注意使用。在实际开发中，应该根据需要选择适当的导出方式。

# 五. vue router 怎么实现输入地址进行路由切换的

Vue Router 实现输入地址进行路由切换的底层原理涉及到 HTML5 History API 和 Hashchange 事件。以下是两种模式的详细说明：

# HTML5 History 模式

在 HTML5 History 模式下，Vue Router 使用 HTML5 History API 实现输入地址进行路由切换。具体实现步骤如下：

  1. 在 Vue Router 的路由配置中，设置 mode 为 'history'，这样就可以开启 HTML5 History 模式。
   const router = new VueRouter({
     mode: 'history',
     routes: [...]
   })

  1. 确保服务器端能够正确地处理路由请求。在使用 HTML5 History 模式时，服务器必须始终返回同一份 HTML 页面，而不是返回 404 或其他错误页面。因此需要在服务器端进行相应的配置，具体方式可以参考服务器相关文档。

  2. 在客户端，Vue Router 使用 window.history.pushState() 和 window.history.replaceState() 方法来修改浏览器历史记录，并在历史记录中添加新的路由记录。当用户在地址栏中输入路由地址时，浏览器会向服务器发送 HTTP 请求，并在页面上显示返回的 HTML。这个过程不会导致页面的重新加载，但是会触发 popstate 事件。

  3. 当用户点击浏览器的后退和前进按钮时，或者在地址栏中输入路由地址时，浏览器会触发 popstate 事件。Vue Router 监听 popstate 事件，并在事件回调函数中根据当前的路由路径进行匹配，以渲染对应的组件。这个过程是通过 VueRouter.prototype.history 方法实现的。

# Hash 模式

在 Hash 模式下，Vue Router 使用 URL 的哈希值来实现输入地址进行路由切换。具体实现步骤如下：

  1. 在 Vue Router 的路由配置中，设置 mode 为 'hash'，这样就可以开启 Hash 模式。

const router = new VueRouter({
  mode: 'hash',
  routes: [...]
})

  1. 在客户端，当用户在地址栏中输入路由地址时，浏览器不会向服务器发送 HTTP 请求，而是在当前页面内切换路由。这个过程不会导致页面的重新加载，但是会触发 hashchange 事件。

  2. 当 URL 的哈希值发生变化时，浏览器会触发 hashchange 事件。Vue Router 监听 hashchange 事件，并在事件回调函数中根据当前的路由路径进行匹配，以渲染对应的组件。这个过程是通过 VueRouter.prototype.hash 方法实现的。

需要注意的是，在 Hash 模式下，需要在路径前添加 # 符号，且该符号后面的内容才是路由路径。在 Hash 模式下，URL 中的哈希值部分不会被包含在 HTTP 请求中，而是只存在于客户端浏览器中。因此，如果在浏览器地址栏中输入带有哈希值的 URL，浏览器不会向服务器发送任何请求，而是直接在客户端进行路由切换。

因为哈希值部分不包含在 HTTP 请求中，所以在 Hash 模式下不需要在服务器端进行额外的配置，可以直接在客户端使用。而且，由于哈希值部分不会影响搜索引擎优化，因此在一些应用场景下，Hash 模式可能更加适合。但是，需要注意的是，在 Hash 模式下，URL 的哈希值部分可能会被视为页面内部的跳转，而不是不同的页面，因此可能会影响一些浏览器的行为（例如在某些情况下，浏览器会把 URL 的哈希值部分自动滚动到页面的对应位置）。

# qiankun 切换子应用示例

在 qiankun 中，可以通过主应用（父应用）控制子应用的加载和卸载，以实现多个子应用之间的切换。具体来说，qiankun 主应用通过加载不同的子应用，从而实现在不同的子应用之间切换。可以通过 loadMicroApp 方法加载子应用，并通过 unloadMicroApp 方法卸载子应用。此外，还可以通过 start 和 stop 方法启动和停止 qiankun。

以下是一个简单的 qiankun 示例，用于加载和卸载一个子应用：

import { registerMicroApps, loadMicroApp, start } from 'qiankun';

// 注册子应用
registerMicroApps([
  {
    name: 'app1',
    entry: '//localhost:8081',
    container: '#subapp-container',
    activeRule: '/app1',
  },
]);

// 加载子应用
loadMicroApp({
  name: 'app1',
});

// 卸载子应用
unloadMicroApp({
  name: 'app1',
});

// 启动 qiankun
start();

在上面的示例中，registerMicroApps 方法用于注册子应用，并指定了子应用的名称、入口地址、容器、激活规则等信息。loadMicroApp 方法用于加载指定名称的子应用。unloadMicroApp 方法用于卸载指定名称的子应用。最后，通过 start 方法启动 qiankun。

需要注意的是，在 qiankun 中，子应用和主应用之间的通信可以通过共享全局变量、使用 window.postMessage 等方式来实现。在子应用中，可以通过 qiankun 对象来获取主应用传递的参数。例如，在子应用中可以通过以下方式获取主应用传递的参数：

const { onGlobalStateChange, setGlobalState, qiankun } = window['qiankun'];

# 七. Vite 原理

Vite 是一个现代化的构建工具，它使用了新的原理来加速应用的开发和构建过程。Vite 的核心思想是基于 ES 模块来构建，而不是像传统的构建工具（如 Webpack 和 Rollup）那样使用打包的方式来处理代码。具体来说，Vite 在开发环境下会使用浏览器原生的 ES 模块加载器来加载模块，而在生产环境下则会使用 Rollup 进行打包。这种方式可以显著提高开发环境下的构建速度和开发体验。

以下是 Vite 的一些核心原理：

  1. 基于 ES 模块

Vite 使用了基于 ES 模块的原理来构建应用。在开发环境下，Vite 会将每个模块作为一个单独的文件来处理，然后通过浏览器原生的 ES 模块加载器来加载模块。这种方式可以避免传统构建工具在开发环境下的重复打包和热更新等问题，从而显著提高了开发环境下的构建速度和开发体验。

  1. 支持 HMR

Vite 支持热模块替换（HMR）技术，在修改代码后可以快速地重新加载和渲染页面，从而加快开发速度。Vite 的 HMR 原理是通过监听文件变化，然后使用浏览器原生的 ES 模块加载器来加载新的模块，然后在页面中使用新的模块来替换旧的模块。

  1. 基于浏览器原生支持的特性

Vite 采用了基于浏览器原生支持的特性来实现构建过程，例如使用浏览器原生的 ES 模块加载器和 HTTP/2 协议等。这种方式可以避免了传统构建工具的一些问题，例如复杂的配置和依赖关系、长时间的构建时间等。

  1. 使用 Rollup 打包

在生产环境下，Vite 使用 Rollup 来进行代码打包。由于在开发环境下已经使用了浏览器原生的 ES 模块加载器来加载模块，因此在生产环境下的打包过程会非常快速和高效。

总的来说，Vite 的核心原理是基于 ES 模块和浏览器原生支持的特性来实现构建和开发过程，从而提高了开发效率和构建速度。

# 八. 引用类型和基础数据类型分别在哪里

基础类型在栈内存，引用类型数据在堆内存，地址指向在栈内存。回答不是面试官想听到的

在 JavaScript 中，基本数据类型有 6 种，分别是：Number、String、Boolean、Undefined、Null 和 Symbol。而引用类型则包括 Object、Array、Function 等等。

基础数据类型在 JavaScript 内存中是存储在栈（stack）中的，当创建一个基础数据类型的变量时，变量名和值都会被存储在栈内存中，当这个变量被复制到另一个变量中时，只是复制了这个变量的值。这些变量在栈中的存储大小是固定的，并且它们的值是不可变的。

引用类型的值则是存储在堆（heap）内存中的，当创建一个引用类型的变量时，变量名和值（即引用地址）都会被存储在栈内存中，而值（即引用地址）所指向的对象则存储在堆内存中。当把一个引用类型的变量赋值给另一个变量时，只是复制了这个变量的引用地址，而不是复制对象本身。由于引用类型的值是存储在堆内存中的，所以它们的大小可以动态改变，并且它们的值也可以被修改。

需要注意的是，在 JavaScript 中，所有的函数参数都是按值传递的，包括引用类型的值。也就是说，当把一个引用类型的值传递给一个函数时，实际上传递的是这个值的引用地址，而不是对象本身。

# 九. 从浏览器输入地址到页面显示发生了什么

在浏览器中输入一个 URL 地址并按下回车键后，会发生以下过程：

  1. URL 解析

浏览器首先对输入的 URL 进行解析，将其分成协议、主机名、端口、路径、查询参数等多个部分。

  1. DNS 解析

浏览器根据解析出来的主机名，向 DNS 服务器发送请求，获取该主机名对应的 IP 地址。

  1. 建立 TCP 连接

浏览器通过 IP 地址和端口号，向服务器发起 TCP 连接请求，与服务器建立连接。

  1. 发送 HTTP 请求

TCP 连接建立后，浏览器向服务器发送 HTTP 请求，请求内容包括请求方法、请求头、请求体等信息。

  1. 服务器响应

服务器收到请求后，根据请求内容生成响应结果，并返回给浏览器。响应内容包括响应状态码、响应头、响应体等信息。

  1. 浏览器解析渲染页面

浏览器接收到响应后，根据响应内容进行解析，构建 DOM 树、CSSOM 树和渲染树。然后根据渲染树中的元素进行布局和绘制，最终将页面渲染到屏幕上。

  1. 关闭 TCP 连接

页面渲染完成后，浏览器关闭 TCP 连接，释放网络资源。

需要注意的是，以上过程是一个简化的描述，实际上还有很多细节和优化。例如，在请求过程中可能会进行缓存、压缩、分片等操作，以提高网络传输效率和响应速度。

# 十. 作用域，请用从底层原理上去解释

作用域是指在程序中定义变量的区域，它决定了变量的可见性和生命周期。在 JavaScript 中，作用域分为全局作用域和局部作用域两种，其中全局作用域是指在代码中任何地方都能访问的变量，而局部作用域则是指在代码块中定义的变量，只能在该块内部访问。

从底层原理上来说，作用域是由 JavaScript 引擎在编译阶段进行静态分析和词法作用域解析来实现的。具体来说，JavaScript 引擎在编译阶段会根据代码的结构和变量的声明情况，将变量的声明和作用域关系建立起来，生成作用域链。作用域链是一个由当前环境和所有父级环境的变量对象（Variable Object）组成的列表，它决定了变量的查找顺序和可见性。

当 JavaScript 引擎在执行代码时遇到变量时，会先从当前环境的变量对象中查找该变量是否存在，如果存在则直接使用该变量的值，否则会沿着作用域链一级一级地向上查找，直到找到该变量或到达全局作用域。如果在全局作用域中仍然找不到该变量，则会抛出 ReferenceError 异常。

需要注意的是，JavaScript 中存在变量提升（Hoisting）机制，即在编译阶段会将变量声明提升到函数或全局作用域的顶部，但变量赋值的过程仍然在执行阶段进行。因此，在函数中使用 var 声明的变量会成为该函数的局部变量，而在函数外声明的变量则会成为全局变量。在使用 let 和 const 声明变量时，由于它们具有块级作用域，因此只能在块内部访问。

# 十一. 前端是如何处理‘百万并发‘的

在前端处理百万并发时，需要考虑多个方面，包括性能优化、负载均衡、缓存优化等。

  1. 性能优化 为了提高前端性能，可以采取多种措施。其中一些常用的措施包括：
  • 减少 HTTP 请求：将多个 JavaScript 和 CSS 文件合并为一个文件，使用雪碧图来减少图片的 HTTP 请求次数，使用字体图标等。
  • 压缩代码：使用压缩工具来减小 JavaScript 和 CSS 文件的大小，以减少页面加载时间。
  • 延迟加载：使用延迟加载和异步加载来减少页面的加载时间，例如使用 require.js 等工具来实现按需加载。
  • 图片懒加载：使用图片懒加载技术来减少页面的加载时间，当用户滚动到图片位置时再加载图片。

  1. 负载均衡 在面对百万并发时，单台服务器可能会无法承受这样的负载，因此需要采用负载均衡技术来将请求分发到多个服务器上进行处理。常见的负载均衡技术包括 DNS 负载均衡、硬件负载均衡、软件负载均衡等。负载均衡可以提高系统的稳定性和可靠性，减少单点故障的风险。

  2. 缓存优化 缓存优化是提高前端性能的重要手段之一。前端缓存可以分为浏览器缓存和服务器缓存两种，其中浏览器缓存包括强缓存和协商缓存两种方式，服务器缓存则包括页面缓存、数据缓存等多种方式。通过合理的缓存策略，可以减少 HTTP 请求次数，降低服务器的负载，提高网站的响应速度和用户体验。

总之，在前端处理百万并发时，需要综合考虑多种因素，包括性能优化、负载均衡、缓存优化等，采用适当的技术手段来提高系统的性能和可靠性。

# 沙盒机制

为了确保不同的前端应用程序之间的安全性和独立性，Qiankun 实现了一个沙盒机制。沙盒机制的目的是将每个应用程序的代码和状态隔离在其自己的运行环境中，以避免应用程序之间的冲突和干扰。

具体来说，Qiankun 在加载每个应用程序时，会将其代码和状态封装在一个沙盒环境中。这个沙盒环境是一个 JavaScript 上下文，它与主应用程序的上下文相互独立，拥有自己的全局对象和作用域链。这样，即使两个应用程序使用相同的变量名或函数名，它们也不会相互干扰。

同时，Qiankun 还通过实现一些安全措施来保证应用程序之间的隔离性。例如，它会禁止子应用程序直接访问主应用程序的全局变量和函数，以避免潜在的安全问题。此外，Qiankun 还提供了一些通信机制，以便不同的应用程序之间可以进行必要的交互。

Qiankun 是一个优秀的微前端框架，它提供了一个集中的路由管理、状态管理和通信机制，以帮助开发人员协调多个应用程序之间的交互。通过使用沙盒机制，Qiankun 可以确保不同的应用程序之间的安全性和独立性。这使得它成为一个强大的工具，可以在不同的前端应用程序之间实现高效的集成和共享。

一个 Web 应用程序，并且您已经有两个不同的代码版本：V1 和 V2。V1 是您最初的版本，V2 包含了一些新功能和修复了一些错误。

# 合并版本

如果您想要合并这两个版本，您可以按照以下步骤进行：
